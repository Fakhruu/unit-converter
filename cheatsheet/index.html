UnitConverter::convert

Since 0.0.1

UnitConverter::convert() â€” Sets the value to be converted.

Description

UnitConverter::convert(mixed $value, int $precision = null): UnitConverterInterface;

Set the unit converters' value to be converted. This method is the first method to be called in the chain of conversion methods.

Parameters

int | float | string $value

The numerical value being converted.

int $precision

The decimal precision to be rounded to

Returns

UnitConverterInterface

Examples

Integer Input

Sets 9,001 units to be configured, represented as an integer.

$converter = UnitConverter::default();
$converter->convert(9001);
Warning: UnitConverter instances configured with the BinaryCalculator can not accept integer input.

Floating Decimal Input

Sets pi-to-the-fifth-decimal-units to be configured, represented as a floating decimal.

$converter = UnitConverter::default();
$converter->convert(3.14159);
Warning: UnitConverter instances configured with the BinaryCalculator can not accept floating decimal input.

Numeric String Input

The converter will generally always accept string input, and is considered the safest type to pass $value in as.

$converter = UnitConverter::default();
$converter->convert('101');
$converter->convert('0.99');

Converting with Precision

The following snippet will convert 5 metres to inches, with a precision of 9 (to the ninth decimal place).

$converter = UnitConverter::default();
$converter->convert(5, 9)->from('m')->to('in'); # (float) 196.850393701
Note: the result of the calculation could have less decimal place values than you specify in the $precision parameter. This will happen if the result cannot be any more precise than what PHP deems.
